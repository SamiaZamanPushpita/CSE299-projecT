# FoodForward Development Plan

## Current Stage
The project proposal has been completed. The next step is to move from planning to implementation in small weekly milestones that match the CSE 299 class schedule.

## Class-wise Development Roadmap

### Class 4 - Update 1
- Prepare the implementation roadmap.
- Draft the first database structure.
- Organize GitHub documentation and team responsibilities.

### Class 5 - Update 2 + Literature Review
- Complete the literature review presentation.
- Initialize the FoodForward Django web project.
- Create the basic homepage and common page layout.
- Prepare the first User, Restaurant, and Charity models.

### Class 6 - Update 3 + Proposed Solution
- Complete the proposed-solution report.
- Implement registration, login, logout, and role handling.
- Prepare separate dashboard entry points for Donor, Restaurant, Charity, and Admin.

### Class 7 - Update 4
- Build Restaurant and Charity profile modules.
- Add meal-package creation and management.
- Add administrator approval for restaurants and charities.

### Class 8 - Update 5
- Build the main donation workflow.
- Allow donors to browse meal packages and select a charity.
- Add simulated payment and donation status tracking.
- Implement the status flow: Pending -> Accepted -> Preparing -> Delivered -> Confirmed.

### Class 9 - Update 6
- Add the "Donate Where It Is Needed Most" rule-based allocation.
- Add notifications and donation history.
- Test the complete workflow and fix major bugs.

### Classes 10-11 - Project Demonstration
- Final bug fixing.
- Prepare sample accounts and demo data.
- Clean the GitHub repository and README.
- Record the one-minute demonstration video.

### Class 12 - Final Presentation
- Final report and final presentation.
- Final GitHub cleanup and submission.

The group will still work feature by feature. For every feature, the frontend, backend, and database work will be connected and tested together before moving to the next feature.
